How Hard is Concurrency?

Conventional wisdom is that concurrent programming is hard:

  1. Sequential programming plus threads, locks, atomicity, ...

  2. The interactions between processes are hard to anticipate, test, and debug.

Maybe the problems arise because concurrency has been added to sequential languages.

The sequential programming in ToonTalk is very limited:

  • no procedures

  • no conditionals

  • no shared data

It is concurrency and communication channels that makes ToonTalk as expressive as other languages.
