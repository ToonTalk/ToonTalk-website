Did ToonTalk Find a Sweet Spot in Programming by Example?

Demonstrative programming systems tend to be either:

  1. Too weak or specialised for general-purpose programming

  2. Clever, but not clever enough, to be a reliable predictable tool

While training a robot in ToonTalk, only a few aspects of the current action are relevant:

  1. The objects involved are either held, in the tool box, in the robot's box, or created or extracted by earlier training actions.

  2. If a part of an object is involved then the semantically relevant aspect is its ordinal position.

  3. If the keyboard is used then which object, if any, is held or being pointed to.

The only relevant actions are picking up, dropping, and sending keyboard characters.

Children rarely create bugs due to a different expectation of how a robot will replay its training. And those that do quickly learn to avoid such mistakes.

Further generalisation is expressed by removing details.
