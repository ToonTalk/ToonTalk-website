How to Create Your Own Demos

WARNING: MUCH OF THIS HAS CHANGED SINCE VERSION 2.33 OR BETA 26

CLICK HERE FOR THE NEW VERSION

Advanced users:

To create a demo just add -o <filename> to the command line before starting ToonTalk. This will create a file named filename.DMO in the "demos" subdirectory of ToonTalk. The "dmo" file records everything you do (every mouse movement, keyboard click, etc.) Double clicking on the file will start up the demo. Alternatively adding -I <filename> to the command line will run the demo. Please check the "demos" subdirectory first to learn the names of existing demos so you don't override them.

If you are creating a demo on a fast computer and expect the demo to be run on slower machines then it is recommended that you add -y 50 or so to the command line while recording to slow down your machine.

Adding subtitles to your demos. When ToonTalk starts running a demo named X it looks for a file named X.ust. ("ust" is the default extension name for (American) English subtitles.) You can override the default for your computer by providing an "SubtitlesSuffix" entry in the "FileExtensions" section of the ToonTalk.ini file in your Windows directory. A subtitle file is made of entries separated by a blank line. Each entry has the time in milliseconds since the beginning of the demo followed by the text of the subtitle. It is recommended that any lines of text more than 60 characters be broken. The "demos" subdirectory of your ToonTalk installation has many sample subtitle files. One way to obtain the millisecond timings is to press F9 while the demo is playing back.This will record the time in your <username>.txt log file in the "users" subdirectory.

Adding narration to your demos. When ToonTalk starts running a demo named X it looks for a file named X.spk. ("spk" is the default extension name for (American) English narration files.) You can override the default for your computer by providing an "SubtitlesSuffix" entry in the "FileExtensions" section of the ToonTalk.ini file in your Windows directory. To create a narration file you should record segments of the narration (256 is the maximum number) and name them "s01.wav", "s02.wav" and so on. These files should be placed on the main ToonTalk directory. While replaying your demo you should press F11 to start the next narration segment. If you prefer to record and synchronize the pre-recorded narration instead then add -d 451 to the command line as well as the -o <filename> and again press F11 when you wish to add the next narration segment. When ToonTalk exits it will write a new "spk" in the demos subdirectory.

Coordinating the subtitles and narration. If you wish to create both narration and subtitles, it is recommended you create the narration first and then run your demo with "-d 3210" which will print the start times of each segment in your <username>.log in the "Users" subdirectory. You can then use these times in your subtitles file.

Tips:

  • A demo file works for the same resolution as it was recorded. With luck you can record a high-resolution demo and copy it to the low resolution subdirectory and it will work. See the subdirectories of "Demos".
  • A demo recorded with speech on might not replay correctly when speech is turned off.
  • Speech is turned off when replaying a demo with narration unless you are running with the text-to-speech engine using DirectSound.
  • If you make a mistake while recording a demo you needn't redo the whole thing. You can run tt202 -I PartlyGood -o NewVersion and then take control just before the old demo goes bad.
  • Avoid using keyboard accelerators for tools (e.g. 'e' to make Dusty erase things). These accelerators differ from language to language so if you use them your demos probably won't work in a different language version of ToonTalk.
  • If you create a demo for a user named Pat, then when that demo is run it may change the user profile for Pat. To avoid this use -t 1 in the command line to indicate that this is a "temporary" user and a user profile should not be kept.
