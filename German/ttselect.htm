What to do if Marty doesn't speak in ToonTalk

If you have installed ToonTalk from a CDROM and are running Windows 95, Windows 98, or Windows NT 4.0, then Marty should be able to speak. Here are some of the things that can go wrong:

  1. No sound card. Not surprisingly Marty can't speak if the operating system can't find the sound card.
  2. Bad sound drivers. The operating system needs programs called sound drivers to run the sound card. Yours may be missing, the wrong version, or too old. Contact your PC vendor or sound card vendor for the latest sound drivers.
  3. 8-bit sound card. If your version of ToonTalk is earlier than 1.04 and your sound card is several years old then it might not be able to deal with 16-bit sound files. This problem has been fixed by the Microsoft speech group in SAPI 4.0 Click here to visit the Microsoft speech group web site to get more information or the latest speech files. Installing Microsoft SAPI 4.0 (Speech API) fixes this problem.
  4. An alternative. Marty speaks using Microsoft's Speech API communicating with Microsoft's DirectX components. If you are running ToonTalk version 1.04 or later, you can try turning this off by adding "-tts_uses_ds 0" to the command line. This will mean you will not hear any sound effects while Marty is speaking. If you are running a version before 1.04, then try "-tts_uses_ds 1" to turn this on.
  5. Click here to report a problem.
