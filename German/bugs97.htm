History of ToonTalk Enhancements and Bug Fixes - 1997

Here is the list of bugs and enhancements in 1997.   Similar lists for 1996, 1998, and 1999 are available as well.

1/7/97

  1. Fixed spelling of twelfth. (Thanks to Jay Schmoll and Kelly Johnson for pointing this out.)
  2. Fixed spelling of CAPITAL in a puzzle. (Thanks to Jay Schmoll and Kelly Johnson for pointing this out.)
  3. Numbers with commas (or in other languages periods) are no longer broken between lines with Marty is talking.
  4. Sometimes Marty broken sentences between talk balloons unnecessarily.
  5. While doing puzzles if you haven't used a bomb then Marty now suggests it much earlier than he used to.
  6. If while doing puzzles you asked Marty to repeat himself and then left before he finished he no longer tries to finish when you come back.
  7. While doing puzzles sometimes if you left while Marty was talking even if he didn't have more to say he tried to repeat himself when you came back.

1/8

  1. Can no longer resize the Start ToonTalk application. (Thanks to Jay Schmoll for suggesting this.)

1/9

  1. While running ToonTalk in full screen mode if you type a Window's command like Alt-Tab or Ctrl-Esc confusion used to result. ToonTalk is now paused and minimized in this situation. (Thanks to Jay Schmoll and Kelly Johnson for pointing this out.)

1/14

  1. Fixed problems with newly introduced sound data type. They can now be placed in boxes and appropriate help is given by Marty.

1/15

  1. Adjusted the size of buttons in the Start ToonTalk program to accommodate languages that need more space.
  2. Fixed a bug that caused the system to crash when Marty was talking about a robot that uses the new built in sound type.
  3. Fixed a bug in puzzle mode which sometimes caused a crash when leaving the house.

1/16

  1. If a puzzle required robots trained in the previous puzzle then if you quit and started again then the robots previously trained were lost.
  2. Fixed the geometry of pictures of the goal for a few puzzles.
  3. Titles no longer trigger the teleport in sound.

1/17

  1. Sometimes copying a nest with things on it led to unpredictable behavior.
  2. If you erased something (made it blank) with Dusty and then put it on the back of a picture or in another house and then put Dusty in reverse it would be restored. Now Dusty only restores blank things that are on the screen.

1/19

  1. If ToonTalk's speed is changed it no longer affects the speed that Marty talks or that subtitles are shown.
  2. Redid part 2 and 3 of the Ping Pong demo and the factorial demo.

1/20

  1. Fixed a recent problem with training robots on boxes that contain covered nests.
  2. Fixed some problems in how Marty gives help in using bombs.

1/23

  1. Marty can talk!!!! (But only in the 32 bit version and currently I'm using a speech engine that I don't have the right to redistribute.) -speak 1 will turn it on -balloon 0 will turn off talk balloons
  2. Scales can now be changed by typing ' ', '+', or '-' if they are tottering or have blanks on both sides of them. (Thanks to Leo Burd for noticing that this feature was broken.)
  3. F1 quickly following F1 was ignored works now. (Thanks to Jay Kahn for reporting this.)
  4. If running with text-to-speech you can shift-click a text pad and it'll be spoken.
  5. You can now have subtitles to demos spoken.

1/26

  1. Some puzzles (like #6) depend upon the previous puzzle. If you set off the bomb in such a puzzle you were left with an empty floor. All you could do is restart ToonTalk and the puzzle would be restored properly. Fixed now.
  2. If you copy Dusty the vacuum you now also copy what's in his stomach.
  3. If a nest is copied or saved and it has 100s of things on top then the system crashed. This happened frequently solving puzzle p42.pzl. (Thanks to Phil Mercurio for reporting this.)

1/28

  1. If -language <name> is missing or misspelled then Versions entry for "Language" in toontalk.ini is used. If that is missing then American English is chosen.
  2. If a DLL is missing for a language the system now tries to get it off the CDROM if possible.

1/28

  1. If -language <name> is missing or misspelled then Versions entry for "Language" in toontalk.ini is used. If that is missing then American English is chosen.
  2. If a DLL is missing for a language the system now tries to get it off the CDROM if possible.

1/30

  1. Fixed various installation problems including one that prevented installing a new version on top of an old one.

1/31

  1. Fixed a problem with the display of subtitles for demos.
  2. Fixed a problem that caused the subtitles to stop half way through the Introductory Tour.

2/7

  1. Sometimes dropping a negative number on a text pad produced odd results.

2/12

  1. Trucks will no longer start to drive if power is off (F8).
  2. F8 won't stop them if they are already running.
  3. You can put a text pad in a truck and it'll end up in the new house.
  4. Fixed a bug when doing puzzles that sometimes crashed when blowing up the house or entering the rocket. (introduced on 1/26)
  5. If you use -puzzle to specify which puzzle and that puzzle depends upon the previous puzzle for the items on the floor of the house then there is nothing there now. Used to be you'd seen the stuff from the last puzzle you did.
  6. If you run demos with narration on a machine without a sound card the system could crash.

2/16

  1. Fixed a bug where when Marty is describing a robot whose thought bubble includes blank boxes it could crash.
  2. If you dropped a truck with stuff inside on the notebook it lost the stuff inside. (Also if you dropped a loaded truck on the stack of trucks in Tooly it was returned.)
  3. When you copied a truck some of the stuff inside wasn't copied.
  4. Robot's now have serial numbers visible on their chests.

2/17

  1. Puzzles that required the saving of very large amount of state behaved erratically. For example, in p42.pzl if the nest contains more than 1000 numbers it used to cause problems.

2/18

  1. Beta 6B released.

3/1

  1. If you copied a picture with a wand in "original" mode then the things on the back were also restored. For example, the Ping Pong ball so it would not bounce.
  2. Adjusted the way mouse movements are mapped to ToonTalk movements to increase sensitivity when window is small and screen is big.

3/6

  1. If a robot was trained holding a magic wand (as is the case for many puzzles) and the wand was set down and later picked up and used then sometimes robots got mixed up.
  2. Robots sometimes held magic wands behind them looked stupid.
  3. You can now put trucks, notebooks, robots, and bombs in boxes. Robots can be trained to use these things.
  4. You can now drop a notebook in a truck and that will be the notebook used by the robots in the new house.
  5. ToonTalk to Java translator now works for everything except pictures, sounds, and remote controls. If you pause ToonTalk while holding a robot or truck, then a corresponding Java applet will be created and displayed.
  6. You can also drop a text pad in a truck and it'll end up in the new house.
  7. If you put a text pad in a truck and then generate a Java applet it will get its name from the text pad rather than generating one.

3/7

  1. Fixed a bug with trucks that are stopped (F8) and fully loaded and animations are finished instantly.
  2. ToonTalk to Java translator can now handle the random number remote.

3/10

  1. Fixed a bug where a robot had a remote control in its thought bubble that has been erased (by Dusty). Sometimes system would crash when blowing up a house with such a robot in it.

3/14

  1. The Toontalk to Java translator now generates comments as well.

3/19

  1. Marty can talk! Only in the 32 bit version though.

3/20

  1. Fixed recently introduced bug where Marty would reject many correct puzzle solutions.

3/21

  1. Can no longer break boxes apart by dropping them on a number while doing puzzles.
  2. When Marty tries to repeat what he was saying before you left the rocket (in puzzles) he now doesn't repeat as much.
  3. Text to speech now works with sound on (though sound is turned off when Marty is around). (Waiting for Microsoft SAPI 3.0 to deal with this well.)

3/22

  1. Marty doesn't repeat himself as much now.
  2. colour palette changed to leave Windows the 20 colors it wants.

3/24

  1. Got Marty's talk balloon and speech synchronized.

3/25

  1. Reduced volume of sound effects when narration is in effect.
  2. Redid the Swedish version of puzzle #41.
  3. Narrations now are in the language ToonTalk is running in unless overridden in toontalk.ini by specifying a value for SubtitlesSuffix. (UST for English and SET for Swedish now.)

3/26

  1. Redid Ping Pong demo part 3.
  2. Robot on the back of pictures no longer run unless the picture is dropped on the floor.

3/27

  1. Pause dialog no longer gives confusing warning about losing stuff not in Notebook.
  2. Speech to text now has an option in the notebook on page 10 for controlling it.
  3. Narration of demos in the 32 bit version sometimes would interrupt itself.

3/30

  1. Fixed bugs in various puzzles.

3/31

  1. Beta Release 7 (and Swedish pre-release 2.0).

4/2

  1. Changed the Java stuff to Java packages.

4/3

  1. Fixed a problem with speech where commas or periods were inside of expressions (like 1,000).
  2. Redid the autostart application (CDROM only) to offer an installation choice on the first screen.
  3. When you leave the rocket speech now stops (and sound effects are restored regardless).
  4. PageDown now works with speech as well. Current sentence is interrupted and Marty skips to the next.

4/7

  1. Greatly improved the synchrnoization between Marty's talk balloons and spoken speech.
  2. While training a robot the robot can now move offscreen a bit used to be you sometimes couldn't pick up something you dropped near the edge.
  3. A robot trained to pick up and drop the whole toolbox got mixed up.
  4. Sometimes a special message from Marty after solving a puzzle was lost. (E.g. sometimes after p16.pzl).
  5. Fixed a bug in the way Marty said Swedish numbers between 1000000 and 2000000.

4/8

  1. Robot serial numbers over 1000 now look better.
  2. Sometimes sound effects were lost if left a room while Marty was talking (out loud).
  3. Sometimes if you start with a particular puzzle (using -puzzle on the command line) then the things you make were placed at the wrong place in the rocket.
  4. Sometimes if you start with a particular puzzle (using -puzzle on the command line) then you would reach a puzzle that depends upon stuff made earlier. Now a warning and advice is given.
  5. If you have a nest in your pocket and a robot is repeatedly giving the bird something then you'll now see the bird deliver the stuff to you and return.
  6. Fixed a problem saving the state of puzzles after puzzle 40.
  7. City is now smaller when running puzzles.
  8. Can no longer cheat during puzzles by "stealing" the magic wand from a robot who has been trained to set it down.

4/11

  1. If you train a robot to drop something in a hole, and then vacuum away (or erase) the whole box from his thought bubble and give him a box with fewer holes you now get a reasonable error message.

4/13

  1. Improved the way ToonTalk internally keeps track of what scene an object is currently part of.
  2. Fixed a bug when blowing up the house in puzzle p53.pzl
  3. Marty now describes boxes with many short sentences rather than one long one.
  4. Fixed a bug that happened when you stood up holding a box while a robot was working on the box.
  5. Fixed a problem displaying large boxes (100s or 1000s of holes).
  6. Set a maximum number of holes for a box since the system slows down terribly as they get to be several 1000 long. 2048 is the max now.

4/14

  1. If you used Dusty in reverse while over a stack (in the toolbox or a notebook) then copies of the stack item were made.
  2. If you dropped the wand and then mouse clicked without moving the mouse sometimes you got the thing last pointed to by the tip of the wand.
  3. In high resolution (-v 25) with a hat sometimes the hat was wrapped around so that the edge of it was behind his head.
  4. If the programmer has the nest in his/her pocket and flies the helicopter the bird now flies there if needed.

4/15

  1. Sometimes the arm wasn't quite attached to the hand properly.
  2. Redid bank account demo.
  3. Changed 32bit implementation so that EXE files are about 1M smaller. (They share the file "resind" with the 16 bit version.)
  4. Text to speech (32 bit only) sometimes turned off sound effects even though Marty didn't appear.
  5. Fixed bugs with text-to-speech and sound effects switching.

4/16

  1. A bird on the floor when given something didn't quite return to where she started but drifted. No longer.
  2. The user profile USR and LOG files now on Win 95 can be over 200 characters long. Previously (and still in Win 3.1) if 2 users shared the first 8 letters they were considered the same user.
  3. If training a robot in a puzzle with a training counter it no longer counts if you just drop or pick up the magic wand.

4/19

  1. Used to be if you dropped something into a truck that was in a box it would crash. Thanks to Katherine Nelson for reporting this.
  2. Sometimes things inside of a box that was on a notebook page didn't display right.
  3. Fixed a new problem where Marty (with speech off) would put several sentences in a talk balloon with a small font.

4/21

  1. Fixed a bug in the 32 bit version where on some machines if you paused and returned some images were clobbered.
  2. On some machines the first Ping Pong demo took a very long time in the titles.
  3. If puzzle files are not installed on the hard disk but are on the CDROM, it now reads them from the CDROM.

4/22

  1. If you are running off the CDROM and your drive is fast enough you can now remove the TempDir entry from toontalk.ini and ToonTalk will no longer load files from the CDROM to temporary hard drive files.
  2. It used to be that if you put the Toolbox down offscreen that you couldn't pick him up again. Now a little bit is always visible and it is graspable.
  3. Marty no longer mentions that text objects that spell a sound can be used to make the sound since this is this way of making sounds is being phased out.
  4. Fixed a problem with puzzle p51 where the number became exactly 3000 rather than larger.
  5. Marty now does a better job of describing the clipboard remote control.
  6. Once again if you change the size or position of the toolbox or the tools that live inside him (Dusty, Pumpy, etc.) then when you restart ToonTalk the tools will be the size and position you indicated.
  7. Sometimes the 32 bit version would lose sound effects if the text-to-speech engine couldn't initialize properly.
  8. If you start up ToonTalk, pause it, click on come back later and then start up ToonTalk again with the same user name then the old ToonTalk is activated.

4/23

  1. Training a robot to put a notebook in a truck caused multiple problems.
  2. Robots got confused if you trained one to put a nest on the floor and then put things in the nest.
  3. If you change the size of the Tool Box then the items inside get a smaller or bigger normal size when retrieved. This is now also true for robots that remove things from the Tool Box.
  4. If you trained a robot to vacuum up the main notebook, or send it off in a truck with a robot that blows up that house then ToonTalk crashed since the main notebook was destroyed. No longer.

4/24

  1. Robot's using magic wands now drop things more accurately.
  2. Marty described scales strangely. Thanks to Lennart and Tobias for pointing this out.

5/7

  1. Fixed a problem where things on top of nests inside of boxes sometimes displayed themselves with the wrong size.
  2. If a robot was contained in something (a box or a truck) which was in a notebook and the robot was trained to get something out of a notebook then he didn't use the notebook he came from. Does now.
  3. Sometimes when the sensor notebook is opened and the keyboard is heavily used then a bit of memory was accessed that shouldn't. Debugging tools caught this but not clear if there are any symptoms.

5/8

  1. The 16 bit version of ToonTalk cannot import pictures that are bigger than 64K and only if they have 256 colors. Marty now gives a warning and nothing is imported.
  2. On a few systems exiting ToonTalk with Alt-F4 caused a Direct Draw error.
  3. If you set off a bomb and then quickly pressed "Esc" to stand up, sometimes the house would not explode until you went back inside. But then Tooly the ToolBox doesn't make it out OK. Fixed now.
  4. Sometimes copying something in the thought bubble of a robot crashes the system. No longer.
  5. A robot trained to drop a box on a number sometimes got confused.
  6. A sensor all by itself on a notebook page was the wrong size.
  7. If you dropped a truck with a robot inside on a notebook sometimes a copy of the robot stayed around until you left the floor.

5/12

  1. Fixed many ToonTalk to Java bugs.
  2. The generated Java code can now be run as an applet or as an application.

5/14

  1. A bird sometimes forgot to deliver something. If it then returned to a house that blew up the item was destroyed improperly. Could lead to a crash.
  2. Fixed various problems associating with shift-clicking text when text-to-speech is available.
  3. Fixed a problem that occurred when more than 128 things were on the screen at once.
  4. Fixed many ToonTalk to Java bugs dealing with trucks.

5/15

  1. When things collided then there was a 50% chance that the sensor would detect it one frame later. So if you wanted them both to react to the collision the delayed one might not still be colliding on the next frame. Thanks to Kevin O'Brien for pointing out this problem.
  2. If 2 objects were moving very quickly relative to their size then they used to be able to pass through each other without a collision detector noticing. No longer.

5/17

  1. Sentence maker demo on the last page of the example notebook on page 6 was enhanced.
  2. Beta 7C released.

5/21

  1. Finished major re-write of Java classes for running translated ToonTalk. No longer heavily uses Java threads. Display of results improved.
  2. 5/22

    1. Fixed a bug where a robot lost track of stuff when something was put in a box, taken out, and then something was put into it.

  3. After training a robot, the limits on how far your arm can go offscreen were not restored so it was easy to move your arm off screen and "lose" it.
  4. If a robot was working somewhere in the city and sending key strokes to the main notebook (looking for a page), then if you picked up the notebook you stopped the robot. Also the robot running elsewhere messed up the size of the items on the first pages of the notebook temporarily.

5/23

  1. Fixed a bug where a notebook shared by lots of robots in different houses is destroyed. (Demo2 of Sentence Maker example could trigger this after waiting long enough in heliocopter while it is running.)
  2. 5/25

    1. Fixed a bug with the sensor for detecting if the control button was down.

    5/26

  3. Fixed a bug where a text pad was sent off in a truck and then the new house with the pad was visited.
  4. Sometimes while exiting ToonTalk while tools are coming out of Tooly system would crash.

5/27

  1. Fix on 5/22 where robot lost track of stuff broke when a robot dropped things on a number (or text pad) on the floor.
  2. Fixed a few problems with ToonTalk to Java translator when magic wand was used more than once without setting it down.

5/28

  1. If a bird comes back to a box to find it now occupied, she'll fly around and wait. If you then got up (or set off a bomb) the system crashed. Also she flew around in a goofy manner.
  2. If a bird is returning to a box that has been vacuumed she now stops.

5/29

  1. If a notebook page contains something that uses collision detection and you leave that page the system used to crash. Problem introduced on 5/15. (Thanks to dcornell for pointing this out.)
  2. Fixed a bug from 5/22 where while training a robot it can be way off-screen.
  3. Fixed a bug where a robot during training picks up a many-holed box and it gets wider and wider.

5/30

  1. Many small changes to the way Marty describes things, especially remote controls.
  2. 6/2
  3. Sometimes Marty when talking about pictures would include a ) when there was no (.

6/8

  1. Translator to Java now supports pictures and sounds. Nearly everything (all the example programs for example) now run fine in a Web browser.
  2. Beta 7D released.

6/11

  1. Java equivalents of ToonTalk programs that use pictures (e.g. Ping Pong) run about twice as fast now.

6/12

  1. Java now supports global sensors for keyboard and mouse.
  2. If a robot was trained with a box with a sensor with a text display (like shift button down sensor) then in the thought bubble it was replaced with the empty string. It is now replaced by a string corresponding to its last state. If this isn't what is desired one can always replace the text in the thought bubble.

6/13

  1. Add new kinds of synthetic images. In addition to rectangles the pictures notebook now contains ovals, lines, rounded rectangles, etc.

6/16

  1. Java now queries the user for values if an empty box hole is matched against something.

6/21

  1. If a picture is a rectangle with pictures on top and its visibility is set to see some then the pictures on top will be visible and the rectangle will not.

6/23

  1. It once again works to drop a flipped picture on top of a flipped picture. The picture underneath will act as if all the robots and boxes have been dropped on it. This is a very useful way of adding or combining behaviors. If you train a robot by giving him boxes containing remote controls they automatically are erased for the thought bubble. This now also works if you remove the box from his thought bubble and replace it. If you flip over a picture with pictures on top then its notebook contains a copy of the pictures on top and a notebook for each one. The copies now have robots and boxes on the back if the original did.

6/27

  1. ToonTalk to Java translator now uses labels on boxes to generate prompts if needed. If a box is empty or contains a nest without a bird and it is matched against some value then the user is asked for a value. The label under that hole is used if there is one. If a bird without nest is given something its nest will be labeled by the label in the hole the bird is in. If there are no labels a generic message is used.

6/30

  1. Fixed a problem with the Ping Pong demos caused by the addition of new pictures.
  2. When text-to-speech is used without talk balloons Marty would sometimes leave before finishing what he was saying.
  3. The remote control for looks now behaves properly when made blank and given a new appearance.
  4. The remote control for looks now becomes a controller of the picture whose back it is dropped on.
  5. Flipped pictures did not always clip the pictures on the back properly so, for example, thought bubbles sometimes stuck out behind the edge of the flipped picture.

7/3

  1. Things on the back of pictures are now automatically laid out in tiling fashion.

7/22

  1. If a robot on the back of a picture loads up a truck with another robot and box then they now end up on the back of the picture as well.
  2. If a robot on the back of a picture loses the box he is working on (e.g. he was vacuumed it up or put it in a truck for another robot to work on) then he also disappears from the back of the picture since he can no longer do anything.
  3. Sometimes if a trained robot picked something up and changed it (e.g. the number of holes of a box) and released it the size would be wrong.

7/23

  1. Redid Ping Pong Demo (part 3).
  2. ToonTalk is now more responsive to mouse movements (maximum speed in various contexts has been reduced).
  3. If Marty needs to say something that he can't speak (an error message from MCI for now) he will switch to using talk balloons for a minute.
  4. Make some error messages more important (so they come through even if Marty's default talk level is increased to 4).

7/29

  1. Beta 8 released

8/1

  1. Fixed a bug with birds and exploding houses (e.g. sentences demo) that crashed system.
  2. Fixed bug in translator to Java when making something empty (e.g. a hole of a box).

  1. Fixed a bug in translator to Java when a notebook has a truck with a robot in it.
  2. Fixed a bug in translator to Java when a robot uses a notebook.
  3. You can no longer vacuum things out of a remote control notebook of a picture. (Thanks to Katherine Nelson for pointing out this problem.)
  4. On puzzle #49 (blow up the other house) if you blew up the wrong house and went back to the rocket the system would crash. No longer. (Thanks to Katherine Nelson for pointing out this problem.)
  5. When in puzzle #62 you flipped the number, the system crashed. No longer. (Thanks to Katherine Nelson for pointing out this problem.)
  6. Sometimes when a robot picked up a box it gradually grew wider and wider. No longer.
  7. When Windows was running with more than 256 colors, then the 16 bit version showed the subtitles as black on black. (Thanks to Katherine Nelson for pointing out this problem.)
  8. When Windows was running with more than 256 colors, when exiting the 32 bit version full-screen mode then errors would occur and leave Windows with 256 colors.

  1. Fixed the release information on the opening screen.
  2. Fixed one of the images of Tooly, the toolbox, settling down. Used to have a purple fringe.
  3. Now when ToonTalk is running in full-screen mode and loses the focus (e.g. Alt-Tab or Ctrl-Esc or another application grabbing it), you now get a specialized dialog for the occasion. Also if Windows was running more than 256 colors, it used to be reset to 256 colors when this happened.

8/6

  1. Sometimes when two robots were working together and the programmer wasn't watching either one, then one of them would lose track of things and stop. Fixed.
  2. Java translator had a problem if a notebook containing a truck containing a robot was encountered.
  3. F9 now only hides your hand and arm but not what you are holding.
  4. Beta 8A released.

8/9

  1. If StartTT.exe has problems launching ToonTalk it now describes the problem in the file c:\StartTT.log
  2. StartTT didn't work right if ToonTalk was installed so that the path name contained too many characters.

8/11

  1. If a problem occurred while running a puzzle then starting again sometimes caused the same problem. Fixed at least for some of the special puzzles that depend upon earlier puzzles.
  2. If ToonTalk is started on a puzzle that requires things from an earlier puzzle but they aren't available it now offers to send you back a few puzzles.
  3. Beta 8A introduced a bug where if ToonTalk wanted to put up a message box, then it would first put up the "pause dialog". Fixed now.
  4. Began using Microsoft Speech SDK 3.0. Text-to-speech handles apostrophes differently so needed to change ToonTalk accordingly.

8/13

  1. If any file is missing from the hard disk, ToonTalk now reads it from the CDROM (if it was installed from a CDROM). It now asks the user to insert the CDROM if it is not.
  2. If an argument to the -language switch is unknown then the first 2 letters are taken to be the country code and files named XXttYY.dll and XXsttYY.dll (where XX is the country code and YY is 16 or 32 depending upon which version of ToonTalk is being loaded) are loaded. This permits a released version of ToonTalk to be localized after the fact.

8/19

  1. Fixed a bug when "taking control" of a demo how you can't pause it properly after that.
  2. Fixed a bug where ToonTalk was trying to ask for the CDROM but didn't have the string library loaded yet.

8/22

  1. Finished complete re-write of installation program. Can install and un-install separate components. Should be more robust and flexible than old one. And it looks more "modern".
  2. Fixed a bug where the toolbox didn't make it out of an exploding house when the power was off. (Thanks to Katherine Nelson for pointing out this problem.)
  3. Fixed a recently introduced bug in reading the history of user. May have caused strange behavior on Windows NT 4.0.

8/23

  1. Many small improvements to the wording of what Marty says.
  2. When Marty describes a sensor that has been operated upon (e.g. "\2" was dropped on it) then he interrupted his description with some advice. The sentence order now makes sense.

8/25

  1. Fixed a problem generating Java applets if a robot flipped something over while holding it.
  2. Fixed problems with some of the comments generated for Java.
  3. Fixed problems with some of the comments generated for Java in Swedish.
  4. Even though most puzzles disable typing of text (too easy to cheat) ToonTalk allowed typing anyway. (Thanks to Tobias Zetterlund for pointing this out.)
  5. ToonTalk also allowed pictures to be flipped even if a puzzle switch disabled this.

8/26

  1. Fixed various bugs that interfered with puzzles 63, 64 and 65.
  2. Changed the policy about pictures (or numbers) with robots on the back. They no longer stop when picked up or dropped in a box.
  3. A box with a picture with robots on the back now starts working when taken out of a notebook.
  4. Puzzle 61 allowed players to bring any number back.
  5. Expanded the hints in puzzle 65.
  6. 8/27
  7. If the argument to -I on the command line contained an extension (.DMO) then it would only work if it was a fully-qualified path name. No longer.
  8. Added 5 more puzzles.

8/28

  1. If a number was flipped over twice, then put in a notebook and then taken out and dropped on a picture the number and the underlying number pad were added to the picture. Now just the number is added.

8/29

  1. Added 4 more puzzles.
  2. After solving the last puzzle you needed to come back to Marty for him to tell you so. If you solved all the puzzles and start "Play Game" then you are now given the option of starting over again.
  3. The command line now permits file names (e.g. puzzle or demo names) to have spaces in them as many Windows 95 files do. Do not use - in names however as that may confuse things.

8/31

  1. Fixed a bug where system crashed sometimes during training a robot during puzzles if the robot picked up the box it was given.
  2. Puzzle #54 now works by waiting for the bird to deliver the message.
  3. Used to be you could make a team of robots some of who haven't been trained yet.
  4. Puzzles that involve results that are dynamic (have robots on the back) are now checked properly. And Marty responds appropriately to different sorts of wrong dynamic answers.
