Fixa Problem med Java Applets gjorda av ToonTalk

Läser du detta i en fil på din hårddisk kan du se den senaste versionen på nätet genom att klicka här.

Mycket kan gå snett när man skapar Java applets.

  1. Du ser webbsidan men den saknar 'Start'- och 'Stop'-knappar. Detta betyder att den java som skapas av ToonTalk inte kompilerar. Om du programmerar i Java kan du se felmeddelanden genom att köra javashow i ToonTalks Java-mapp. Rapportera gärna sådana problem till support@toontalk.com. Skicka med denToonTalk-sak som skulle sparas i Java genom att trycka control-c medan du håller i saken och klistra sedan in den i din epostrapport.

  2. Java applet stannar när du tryckt på 'Start'-knappen. Det finns exempel på att detta hänt i en viss webbläsare och ett visst operativsystem men fungerat i andra. Prova i andra webbläsare. Rapportera i vilket fall som helst gärna till support@toontalk.com. Om du kan, skicka i form av en javalog.txt file eller i Java console. När man för musmarkören över en applet kan webbläsaren ibland visa ett felmeddelande i statusraden.

  3. Java applet körs men bilderna visas bara som'?' och vissa ljud hörs inte. Om du använder egna bilder i ToonTalk så kräver Java att de är i GIF eller JPG. Skulle bilderna vara i BMP finns det hundratals bildprogram som kan konvertera bilden och spara den som GIF eller JPG och spara i samma mapp eller enligt sökvägen. JPG brukar vara bättre för foton och GIF för bilder som ska ha partier som är genomskinliga.

  4. Java applet körs men det är en gammal applet, inte den som ToonTalk skapade. Om du kör Internet Explorer prova med att hålla ner ctrl-tangenten och samtidigt trycka på knappen för Uppdatera.

  5. Java applet körs men den slutar hänga med musrörelserna. Java applets kan inte köra musrörelser så ibland slutar ToonTalk-spel som använder musen att fungera när de körs i Java. Problemet består i att musen har hamnat utanför spelplanen. Du kan flytta tillbaka den och ordna upp det hela. Ofta vill man flytta musen undertill och komma tillbaka från en annan sida. Rekommendationen är att spelen använder musrörelser eller tangentstyrda kommandon för att undvika problemet.

  6. Netscape 6 visar spelet i konstiga färger och det går väldigt långsamt. Detta har bara observerats när Windows körts i 256 färger. Ändra till 16-bitars, 24-bitars, eller 32-bitars färger så löser det problemet.

  7. Java applet går bra men vissa ljud hörs inte. Hör du inte något ljud är chansen stor att det är något annat program, kanske ToonTalk, som håller filen i ljudsystemet. Om programmet som håller ljudet avslutas, kan det lösa problemet.
    En annan tänkbar orsak är att ljudfilen saknas eller är i fel format. De flesta webbläsare kan bara läsa AU-filer som är samplade i 8012 hertz. (Netscape 6 är uppenbarligen inte så kinkig.) Det finns många program kan läsa en WAV-fil och göra den till rätt sort.

  8. Webbläsaren hänger sig eller lämnar felmeddelande. Vissa versioner av Microsoft Internet Explorer (version 4.0 SP1 - Windows 98 och vissa versioner av 5.0 likaså) visar "Invalid Page Fault" den ska köra en ToonTalk Java applet.
    För att komma förbi detta problem kan du skaffa en nyare version av webbläsaren. Denna bugg i Internet Explorer är borta i version 5.5 och senare.

  9. Du ser ett meddelande om att det är ett säkerhetsproblem att använda BMP-filer. Detta betyder att Java använder BMP-filer eftersom GIF-filerna saknas. Javaprogrammet kommer ändå köras. Vill du att det ska fungera i en webbläsare får du skapa GIF-filerna och göra om applet.

Vill du lägga upp Java applets från ToonTalk på en webbserver kan du koiera filerna i Java-mappen i ToonTalk-mappen till webbservern. system.zip och javashow.bat behöver inte kopieras. Du kan avgöra vilka class filer som behövs genom att se på tiden när de skapades eller genom att kolla källfilen. Från version 2.35 or beta 27 kommer din loggfil (i Users-mappen i ToonTalk med ditt namn följt av ".txt") att lista ljud och bilder som applet använder.
