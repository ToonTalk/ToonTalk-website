Creating a Default User in ToonTalk

ADVANCED USERS ONLY.

You can customize ToonTalk so that all new users begin as "copies" of a user you have created. You can create a user and customize things, change the main notebook, or save the city and everyone will start off with those changes.

There are two ways of telling ToonTalk about the default user:

  1. Adding an entry to toontalk.ini. If you set DefaultUser then every new ToonTalk user on that computer will be affected.
  2. Adding to the command line (typically of an icon). If you add "-default_user username" to the command line of the ToonTalk icon then every new user who starts ToonTalk using that icon will be affected. You may want to create multiple icons with different default users for different purposes.

Note that if the main notebook of your default user contains user pictures or sounds that are not also in the default user's saved city then you need to set SaveMediaInMainNotebook=1 in toontalk.ini.

Also note that if your default user has a saved city and the user logs in with a user name that already has a saved city they will end up in the their saved city, not the saved city of the default user. You can avoid this by using -must_be_new_user 1 on the command line.

Tips

  • Pick a user name for your default user that a user won't pick accidentally. Anyone logging with the default user's name will change things for all future new users.
  • If you save a city while logged in as the default user then everything is saved including things like the contents of Dusty's stomach. One way to save a city with nothing in Dusty's stomach is to build a temporary house, go there, have Dusty spit out everything, and then have a robot destroy the house after you have left the house. If the robot destroys the house while you are in then Dusty will suck everything up again.
  • It is OK to add to the main notebook but removing or replacing things can cause confusion. There may even be robots that were trained to, say, find a sensor in the sensor notebook on page 4. If you replace page 4 with something else then that robot will break.
  • You might want to delete the USR file of the default user since this contains information like what things Marty has already explained and what puzzles you have already done. On the other hand if you want to customize how Marty speaks or how much he speaks using the Options notebook then leave the USR file alone.
